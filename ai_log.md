# AI Log — Weather Delivery Delay Script

This document lists the AI prompts used during the development of `main.py`.

---

## 1. Parallel Fetching Logic

**Prompt used:**
> "I have a list of orders in a JSON file. Each order has a city. I want to fetch the current weather for each city using the OpenWeatherMap API concurrently in Python — not one by one. Show me how to do this using asyncio and aiohttp with asyncio.gather()."

**What it helped build:**
- `fetch_weather(session, city)` — async function to call the API
- `main()` — uses `asyncio.gather(*tasks)` to fire all requests at the same time
- `aiohttp.ClientSession` as the shared HTTP session across all concurrent calls

---

## 2. Error Handling Logic

**Prompt used:**
> "One of my cities in the JSON is invalid (InvalidCity123). The OpenWeatherMap API returns a non-200 status for it. I want to log the error for that city but NOT crash the script — the other valid cities should still be processed. How do I handle this in an async Python script?"

**What it helped build:**
- `try/except ValueError` block inside `process_order()`
- Raising a `ValueError` in `fetch_weather()` when API status is not 200
- Logging the error with `logging.error()` and setting status to `"Error - City Not Found"`

---

## 3. Weather-Aware Apology Message (AI Challenge)

**Prompt sent to Gemini at runtime (generated dynamically per order):**
> "Write a short, warm apology message for a customer whose delivery is delayed due to weather. Customer name: {customer_name}. City: {city}. Weather condition: {weather_condition}. Keep it under 2 sentences and start with 'Hi {customer_name},'"

**Example output generated by Gemini:**
> "Hi Alice, your order to New York is delayed due to Rain and we sincerely apologize for the inconvenience. We appreciate your patience and are working hard to get your delivery to you as soon as conditions improve!"

**What it helped build:**
- `generate_apology()` function that calls Gemini API dynamically per delayed order
- Personalized messages stored as `apology_message` field in the updated `orders.json`
